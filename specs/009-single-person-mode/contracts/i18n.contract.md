# Contract — i18n Keys (EN + zh-TW)

**Scope:** New user-visible strings introduced by feature 009. Governed by Constitution Principle VII (Bilingual First-Class — NON-NEGOTIABLE): every key ships both translations in the same commit and lands in `FIRE-Dashboard Translation Catalog.md` together with the code change.

All keys go into the Generic dashboard only (`FIRE-Dashboard-Generic.html`). No changes to `FIRE-Dashboard.html` (per FR-029).

## 4. Wiring rules (Principle VII enforcement)

For every key above:

1. Add to `TRANSLATIONS.en` AND `TRANSLATIONS.zh` in `FIRE-Dashboard-Generic.html` in the same commit that introduces the DOM hook.
2. Mirror the key + both strings into `FIRE-Dashboard Translation Catalog.md`.
3. Never interpolate English directly into a JS template literal that renders to the DOM — use `t(key)` or `t(key, ...args)`.
4. When the filing-status display is rendered dynamically (§2), `switchLanguage()` must trigger a re-render so the label flips on language toggle (existing behavior of the tax-planning section; verify not broken).

---

## 5. Audit checklist (Manager-side, pre-merge)

Run against each PR commit that touches `FIRE-Dashboard-Generic.html`:

- [ ] Every new DOM node with user-visible English text has a `data-i18n` attribute.
- [ ] Every new `data-i18n` key is present in BOTH `TRANSLATIONS.en` and `TRANSLATIONS.zh`.
- [ ] Every new key is mirrored in `FIRE-Dashboard Translation Catalog.md`.
- [ ] No string contains hardcoded English in a JS template literal that reaches the DOM.
- [ ] `grep -oE '[A-Za-z]{4,}'` on the changed HTML fragment — every 4+ char word is either inside `TRANSLATIONS`, behind `data-i18n` / `t()`, or on the Exemption list (proper names, FIRE-specific terms, industry acronyms, currency, country ISO codes, emoji).

Any PR failing this audit is rejected under Principle VII and must be fixed before merge.

---

## 6. Non-goals

- Translating existing strings already covered by `FIRE-Dashboard Translation Catalog.md` — out of scope, other features own those.
- Adding a third language — out of scope.
- Adding translated country names for scenarios — out of scope (already covered by earlier features).
